Project Brief

Background

There is an existing new hire onboarding experience to help new hires fill out important paperwork, get familiar with the team and work location, etc. However, it is designed for desktop and therefore the mobile experience is not user-friendly.

So, this project is to identify usability problems for the mobile new hire onboarding experience and improve it.

Target User

The target user is general employees who can be of any age, any gender, or any profession.

Process & Plan

Goal

Improve usability of the new hire onboarding experience on a mobile device such as making the design easy to navigate, tap, etc.

02/04

Pain Points

What's the original new hire experience?

The video below shows the existing onboarding experience.

I conducted an informal heuristic review and identified 5 types of usability problems:

Paint Point 1

Confusing Double Menu

The mobile app has already had a menu at the upper left corner for the main navigation. Keeping another menu icon for the new hire onboarding wizard will cause confusion.

Paint Point 2

Messy View

The width of certain components will exceed the screen width and therefore look messy on mobile.

Paint Point 3

Can’t Enter Information

The original new hire onboarding experience doesn’t support the keyboard and therefore users can’t enter information.

Paint Point 4

Not Finger-Friendly

Certain components are too small to tap.

Paint Point 5

Painful Main Navigation

When the page is long, users have to scroll up/down a lot to navigate to the previous/next page.

03/04

Design Iterations

Design Process

As shown in the process chart at the top, the goal of this project is to simply improve usability, and therefore there isn’t an ideation phase. After identifying the usability problems, I directly created the hi-fi prototype with Axure.

We’ve tested the usability in late May and all participants can complete all tasks.

Design Highlight on Key Paint Points

Design Solution on Paint Point #1

Paint Point 1 Confusing Double Menu

Before

Confusing double menu icons for main and sub-navigation.

After

Cleaned up the menu icon and use the wizard to present the new hire process.

Design Solution on Paint Point #2 & #3

Messy View & Can’t Enter Information

Before

Messy look due to restricted component width.
No keyboard support.

After

Choose mobile-friendly components and keep the view neat and clean.
Support keyboard.

Design Solution on Paint Point #4

Paint Point 4 Not Finger-Friendly

Before

Certain button/link are too small to tap.

After

Resize them and make it easier to tap.

Design Solution on Paint Point #5

Paint Point 5 Painful Main Navigation

Before

A lot of scrolls when the page is long.

After

Easy access to main navigation regardless the length of the page.
